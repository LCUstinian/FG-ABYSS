**如果你是用 TRAE CN 阅读本文档，请先理解以下关键点**：

### 核心架构

```
FG-ABYSS = Tauri V2 桌面应用 (Rust + Vue 3 + TypeScript)
├── 前端 (src/)        : Vue 3 组件 + Pinia 状态管理
├── 后端 (src-tauri/)  : Rust 核心逻辑 + Tauri Commands
```

### 关键功能边界

| 模块 | 路由 | 功能 | 是否远程 |
|------|------|------|---------|
| **载荷管理** | `/payload` | 生成 WebShell 代码 | ❌ **本地** |
| **项目管理** | `/project` | 管理远程 WebShell | ✅ **远程** |
| **控制台** | 弹窗 | 操作远程服务器 | ✅ **远程** |

### 重要概念

- **载荷 (Payload)**: 生成 WebShell 代码（本地操作，不接触远程）
- **WebShell**: 远程服务器上的后门文件（远程操作）
- **项目 (Project)**: 管理多个 WebShell 的容器
- **控制台 (Console)**: 右键 WebShell 打开的窗口，以 TAB 展示插件功能

### 开发优先级

1. **P0**: 载荷生成、项目管理、加密通信
2. **P1**: 文件管理、数据库管理、终端管理（插件）
3. **P2**: 插件系统扩展

## 重要概念说明

**在阅读本文档前，请先理解以下核心概念**：

### 📄 WebShell (Shell)
- **定义**: 远程服务器上的后门文件
- **示例**: `test.php`, `shell.jsp`, `backdoor.aspx`
- **特点**: 部署在目标服务器上，用于远程控制

### 🎯 载荷 (Payload)
- **定义**: 生成 WebShell 的代码/脚本
- **示例**: PHP 代码、JSP 代码
- **特点**: 本地生成，可复制/下载，**不涉及远程操作**

### 📁 项目 (Project)
- **定义**: 管理多个 WebShell 的容器
- **用途**: 按渗透项目组织 WebShell
- **示例**: "项目 A" 包含 5 个 WebShell

### 📦 插件 (Plugin)
- **定义**: 扩展功能的模块
- **类型**: 文件管理、数据库管理、终端管理等
- **运行**: 在控制台窗口中以 TAB 形式展示

### 🖥️ 控制台 (Console)
- **定义**: 操作远程 WebShell 的窗口
- **打开方式**: 右键 WebShell → 连接控制台
- **展示**: 并排 TAB 展示多个插件

## 1. 产品概述

### 1.1 产品定位

FG-ABYSS 是一款现代化的 WebShell 管理工具，用于渗透测试人员管理和使用 WebShell。产品基于 Rust + Tauri V2 + Vue 3 技术栈构建，提供跨平台、高性能、安全可靠的 WebShell 管理能力。

### 1.2 目标用户

- **渗透测试工程师**: 需要管理多个 WebShell 的专业安全人员
- **红队成员**: 执行红队行动，需要隐蔽、高效的工具
- **安全研究员**: 研究 WebShell 技术和防御方案的研究人员

### 1.3 核心价值

1. **跨平台**: 支持 Windows、macOS、Linux，一次构建多平台运行
2. **高性能**: Rust 后端提供系统级性能，内存占用低
3. **安全性**: 多层安全防护，加密存储和通信
4. **现代化**: Vue 3 + TypeScript 前端，现代化 UI/UX
5. **可扩展**: 插件系统支持功能扩展
