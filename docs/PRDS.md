软件需求规格说明书 (SRS)
项目名称：FG-ABYSS (非攻 - 渊渟)
版本号：v1.0.0
文档状态：草案
密级：内部保密
1. 引言 (Introduction)
1.1 产品定位
FG-ABYSS 是一款基于 Rust + Tauri 架构开发的下一代跨平台桌面端 WebShell 管理与载荷生成工具。旨在提供高隐蔽性、强对抗能力及模块化的红队作战支持，对标并超越现有开源工具（如哥斯拉），解决传统工具在流量特征、混淆强度及扩展性上的痛点。
1.2 技术栈架构
核心框架: Rust (后端逻辑), Tauri (桌面容器)
前端界面: Vue 3 + TypeScript + Naive UI
数据存储: SQLite (本地加密存储)
关键依赖:
混淆引擎: Tree-sitter (多语言 AST 解析)
加密算法: AES-256-GCM (载荷), ChaCha20-Poly1305 (日志), Ed25519 (插件签名)
2. 总体设计 (System Overview)
2.1 界面布局规范
系统采用 左右分栏式布局：
左侧导航栏 (Sidebar)：固定宽度，包含五大核心模块入口（首页、项目、载荷、插件、设置）。
右侧内容区 (Main Content)：动态渲染对应模块的功能视图。
自定义标题栏 (Custom TitleBar)：集成主题切换、语言国际化快捷入口，隐藏原生系统标题栏以适配深色模式。
底部状态栏 (Status Bar)：实时监控系统资源（PID, CPU/MEM 占用）、当前连接状态及全局代理信息。
2.2 数据编码与通信规范
文件默认编码: UTF-8 without BOM
载荷通信协议链: 原始数据 → Hex 编码 → 特殊字符 URL-encode → Base64 编码
默认加密套件: AES-256-GCM (预留算法插拔接口)
3. 功能需求 (Functional Requirements)
3.1 首页模块 (Dashboard)
注：作为非核心监控面板，优先级的实施顺序可后置。
数据概览: 可视化展示项目总数、WebShell 存量、插件数量及载荷类型分布。
风险热力图: 基于地图或列表形式，展示各项目中 Shell 的存活率、最后活跃时间及 WAF 拦截频次统计。
系统监控: 实时显示当前进程资源占用情况。
快捷操作: 提供高频功能（新建项目、快速生成载荷）的入口。
活动审计: 滚动展示最近的关键操作历史。
3.2 项目管理模块 (Project Management) - 核心
3.2.1 项目生命周期管理
创建 (Create):
必填项：项目名称、描述。
系统字段：自动生成 UUID、时间戳 (Created/Updated/Deleted)。
维护 (Update/Delete): 支持编辑信息及软删除（标记删除状态，不物理移除数据）。
检索 (Query): 支持多条件筛选的项目列表展示。
回收站机制:
独立视图展示已软删除项目。
支持单条恢复或批量恢复。
数据导出: 支持导出单个或全部项目配置及其关联的 WebShell 信息（加密格式）。
3.2.2 WebShell 实例管理
实例创建:
前置条件：必须归属于特定项目。
配置项：URL、密码、编码、代理配置（类型/主机）、连接模式（文件/内存）、脚本类型、备注。
系统字段：UUID、所属项目 UUID、存活状态标志位。
功能验证：内置“测试连接”功能，验证连通性后保存。
实例维护:
列表交互: 支持列宽拖拽、分页、全文搜索。
右键上下文菜单: 快速执行控制、清理缓存、编辑、删除、导出配置。
回收站: 独立的已删除 Shell 列表，支持单条/批量恢复。
3.3 载荷工厂模块 (Payload Factory) - 核心
3.3.1 智能载荷生成
连接模式选择:
文件类: 通用型 (类哥斯拉)、代理型 (复用 Suo5)、混合型。
内存类 (二期规划): Java/.Net 内存马 (通用/代理/混合)。
脚本类型适配: 根据模式自动过滤支持的脚本语言 (PHP, JSP, ASPX, ASP)。
高级混淆引擎 (基于 Tree-sitter):
L1 轻量级: 变量随机化、字符串拆分、函数名替换 (AST 局部替换)。
L2 标准级: 控制流扁平化、垃圾代码注入、表达式重组 (AST 结构性修改)。
L3 高对抗级: 敏感 API 反射调用、闭包模拟、动态密钥派生 (语义级重写)。
多语言支持: 统一中间件适配 PHP, Java, C# 语法树。
流量伪装: 支持模拟 JSON、Form-Data、图片上传、常规 API 请求等合法业务流量特征。
输出配置:
密码策略：自定义或随机生成 16 位强密码。
文件名策略：去特征化命名（伪装为正常资源文件）。
预览与交付: 代码高亮预览、一键复制、加密下载。
3.3.2 载荷资产库
历史记录: 表格化展示生成历史（文件名、大小、类型），支持元数据保存。
模板管理: 支持用户自定义基础 WebShell 模板的新增与维护。
3.4 插件生态模块 (Plugin Ecosystem)
内置插件集:
命令执行: 伪终端 (PTY) 模拟，提升交互体验。
文件管理: 断点续传、哈希校验、时间戳篡改。
数据库管理: 自适应识别 MySQL, MSSQL, Oracle 等环境。
本地插件加载: 支持从指定目录动态加载/卸载第三方插件，具备启停控制。
插件商店:
安全机制: 强制 Ed25519 数字签名验证，杜绝恶意代码注入。
分发能力: 在线浏览、校验并下载经过签名的插件。
3.5 系统设置模块 (Settings)
外观定制: 深色/浅色主题切换、中/英国际化、全局强调色自定义 (10 种预设 + 取色器)。
网络连接:
全局策略: 超时阈值、重试次数、User-Agent 轮询池。
代理配置: 支持全局 HTTP/SOCKS5 代理；支持单 Shell 独立代理覆盖。
关于软件: 版本信息、作者介绍、开源协议声明。
4. 非功能需求 (Non-Functional Requirements)
4.1 安全性 (Security)
审计日志:
所有敏感操作（生成、连接、命令执行、插件加载）必须记录。
存储介质：加密 SQLite 表 audit_logs。
加密方式：日志内容使用 ChaCha20-Poly1305 加密，密钥由用户主密码通过 KDF 派生。
插件沙箱: 插件运行需经过严格的签名校验，防止未授权代码执行。
内存安全: 利用 Rust 语言特性，确保无内存泄漏及缓冲区溢出风险。
4.2 性能与可靠性
响应速度: 界面操作响应时间 < 200ms，大规模列表（>1000 条）渲染无明显卡顿。
稳定性: 支持长连接保持，网络波动时具备自动重连机制。
兼容性: 完美支持 Windows, macOS, Linux 主流发行版。
4.3 可扩展性
模块化设计: 载荷生成器、加密算法、插件接口均采用抽象层设计，便于后续迭代（如增加新的加密算法或脚本语言）。
5. 数据字典与隐含逻辑 (Data & Logic)
表格

6. 开发建议与路线图 (Roadmap Suggestions)
Phase 1 (MVP): 完成核心架构搭建，实现项目管理、基础 WebShell 连接、文件类载荷生成 (L1 混淆)、内置基础插件。
Phase 2 (Advanced): 集成 Tree-sitter 实现 L2/L3 高级混淆，完善内存马生成逻辑，上线插件商店及签名验证机制。
Phase 3 (Polish): 优化 UI/UX，完善审计日志加密存储，进行多平台兼容性测试及安全渗透测试。
