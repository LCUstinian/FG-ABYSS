## 重要概念说明

**在阅读本文档前，请先理解以下核心概念**：

### 📄 WebShell (Shell)
- **定义**: 远程服务器上的后门文件
- **示例**: `test.php`, `shell.jsp`, `backdoor.aspx`
- **特点**: 部署在目标服务器上，用于远程控制

### 🎯 载荷 (Payload)
- **定义**: 生成 WebShell 的代码/脚本
- **示例**: PHP 代码、JSP 代码
- **特点**: 本地生成，可复制/下载，**不涉及远程操作**

### 📁 项目 (Project)
- **定义**: 管理多个 WebShell 的容器
- **用途**: 按渗透项目组织 WebShell
- **示例**: "项目 A" 包含 5 个 WebShell

### 📦 插件 (Plugin)
- **定义**: 扩展功能的模块
- **类型**: 文件管理、数据库管理、终端管理等
- **运行**: 在控制台窗口中以 TAB 形式展示

### 🖥️ 控制台 (Console)
- **定义**: 操作远程 WebShell 的窗口
- **打开方式**: 右键 WebShell → 连接控制台
- **展示**: 并排 TAB 展示多个插件

## 1. UI 设计概述

### 1.1 设计理念

- **简洁高效**: 界面简洁，操作高效
- **现代化**: 使用现代化的设计语言
- **一致性**: 统一的视觉语言和交互规范
- **响应式**: 自适应不同屏幕尺寸

### 1.2 技术栈

- **框架**: Vue 3 + TypeScript
- **组件库**: Naive UI 2.38+
- **状态管理**: Pinia 2.1+
- **路由**: Vue Router 4.3+
- **国际化**: vue-i18n
- **主题**: Naive UI 内置主题系统

### 1.3 设计原则

1. **少即是多**: 减少视觉干扰，突出核心功能
2. **渐进式披露**: 按需展示信息，避免信息过载
3. **即时反馈**: 用户操作后立即反馈
4. **容错性**: 允许用户犯错，提供撤销/恢复机制

### 4.2 语言切换组件

```vue
<!-- src/components/layout/LanguageSwitcher.vue -->
<template>
  <n-dropdown :options="localeOptions" @select="handleSelect">
    <n-button quaternary>
      <template #icon>
        <n-icon :component="GlobeOutline" />
      </template>
      {{ currentLocaleName }}
    </n-button>
  </n-dropdown>
</template>

<script setup lang="ts">
import { computed } from 'vue'
import { useI18n } from 'vue-i18n'
import { GlobeOutline } from '@vicons/ionicons5'

const { locale } = useI18n()

const localeOptions = [
  { label: '🇨🇳 简体中文', key: 'zh-CN' },
  { label: '🇺🇸 English', key: 'en-US' },
]

const currentLocaleName = computed(() => {
  const lang = localeOptions.find(l => l.key === locale.value)
  return lang?.label.split(' ')[1] || '简体中文'
})

const handleSelect = (key: string) => {
  locale.value = key
  localStorage.setItem('locale', key)
}
</script>
```
